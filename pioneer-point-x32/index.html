1Your rig — why it's the robust choice

Analog splitter snake → two independent X32s.

This matters more than it sounds like it does. The alternative — sharing an S16/S32 stage box over AES50 — means the two desks share preamps. So when your monitor engineer rides gain to kill feedback, your FOH mix moves too. It's a classic X32 festival trap.

With an analog split:

  • ✓ Each console has its own preamps — independent gain, no interference
  • ✓ No AES50 clock/sync between desks — nothing to misconfigure, nothing to drop out
  • ✓ Either desk can crash or reboot and the other one keeps working

The only thing the two desks physically share is the mic signal and phantom power — which is exactly what the next section is about.

2Phantom power — settle it before doors

⚠ The #1 thing that bites analog-split festivals

A splitter snake has a direct leg (hard-wired) and isolated leg(s) (transformer). Phantom power (+48V) only passes through the direct leg — transformers block DC.

So, before doors:

  1. Find out which console is on the direct leg of your splitter.
  2. Only that console sends +48V. The other leaves phantom OFF on every channel.
  3. Tell your monitor engineer explicitly. Agree out loud. Don't assume.

If phantom is on the wrong leg: condensers and active DIs go dead or weird, and you'll waste twenty minutes at line check chasing a "bad channel" that isn't bad.

If both desks send +48V: at best it's redundant; at worst you get noise or a fight between supplies through the split.

3No audio — the fast tree

Work top to bottom. Don't skip — the answer is almost always higher up than you think.

  1. Is signal hitting the desk at all?
    • Meter on the channel strip — any movement?
    • Gain up? Phantom on (if it's a condenser/active DI, and you're on the direct leg)?
    • No meter → the problem is before the console: cable, mic, splitter, stage.
  2. Is the channel getting the right source?
    • ROUTING → Inputs — is the block set to Local (your XLR ins)? Not Card, not AES50.
    • Channel Config → is Source the input you think it is?
  3. Is it muted somewhere you're not looking?
    • Channel mute
    • DCA mute — a muted DCA silences its members even with the channel unmuted
    • Mute group — the sneaky one
  4. Is it going to the main bus?
    • Channel Home → Main LR assign is ON
    • Channel fader actually up
  5. Is Main LR getting out of the box?
    • Main LR fader up, not muted
    • ROUTING → Out 1-16 — is Main L/R actually patched to the physical XLR outs you're using? This is the one that gets people: meters dance on the desk, nothing comes out of the PA.
  6. Still nothing?
    • Solo/PFL stuck on somewhere? (Check the solo clear.)
    • Headphones work but XLR out doesn't → it's an output patch problem, guaranteed. Back to step 5.

4Monitor desk

For your monitor engineer.

Sends must be PRE-FADER

That's the whole point — so when you pull the lead vocal down at FOH, the drummer's wedge doesn't change.

  • Channel sends to the monitor buses: pre-fader
  • Bus output tap: post-fader, so the bus master fader still controls wedge volume
  • Limiter on every monitor bus, threshold below −3 dBFS — cheap insurance against a dropped mic or a feedback squeal going straight into someone's ears
  • Sends on Fader is the fast way to build a mix: hold the bus's select button, then the faders become that bus's sends. This is how monitor mixes get built quickly at line check.

5Scenes & safes — the festival killer

Multiple bands = multiple scene recalls = the chance to nuke your PA between sets.

When you load a band's scene, by default it can overwrite routing, output patch, and preamp gains. Load the wrong scene and the house goes silent — mid-changeover, with an audience.

Two kinds of safes, and the difference matters

TypeScopeUse
Scene Safes ⚠ Per-scene, NOT global Set on each scene individually. Easy to forget one.
Channel / Parameter Safes ✓ Global — apply to every recall This is what you want for festival protection.

Set global safes before doors

  • Routing I/O — so a scene recall can't repatch your inputs
  • Output Patch — so a scene recall can't disconnect your PA
  • Preamp (HA) — so a scene recall can't blow up your gains
  • Consider safing mutes on input channels, so a stale scene can't mute something live

Then a band's scene recalls their channel processing (EQ, comp, mix) without ever touching the infrastructure keeping sound in the room.

Between bands

  • Save a scene per band, named clearly
  • Save the show file to USB — and do it again after you finish tweaking at soundcheck
  • If a recall goes wrong: don't panic-fix twenty things. Reload your known-good scene.

6Before you leave the house

  • Both X32s on the same firmware (mismatches cause weird show-file behavior)
  • Show file saved to USB, and a second USB with the same file
  • Agreed with monitor engineer: who sends phantom (the direct-leg desk)
  • Identical channel order on both desks — so "gtr 2 is ringing in wedge 3" means the same thing to both of you
  • Spare XLRs, spare USB, headphones
